根据您提供的系统功能模块实现流程报告，我制定了以下项目基础框架搭建计划。本项目将采用 **前后端分离** 架构，前端使用 Vue3 + TypeScript + Vite，后端使用 Flask。

## 目录结构规划
我们将严格按照报告推荐的目录结构进行搭建：

```text
rumor-labeling-system/
├── frontend/                # Vue3 前端项目 (pnpm + Vite)
│   ├── src/
│   │   ├── services/        # API 接口封装 (Axios)
│   │   ├── views/           # 页面组件
│   │   ├── components/      # 通用组件
│   │   ├── router/          # 路由配置
│   │   ├── store/           # 状态管理 (Pinia)
│   │   └── ...
│   └── ...
├── backend/                 # Flask 后端项目
│   ├── app/
│   │   ├── routes/          # 路由蓝图 (Blueprints)
│   │   ├── models.py        # 数据库模型 (SQLAlchemy)
│   │   ├── config.py        # 配置文件
│   │   └── __init__.py      # 应用工厂
│   ├── run.py               # 启动脚本
│   └── requirements.txt     # Python 依赖
└── ...
```

## 实施步骤

### 第一阶段：后端环境搭建 (Backend)
1.  **创建目录结构**：建立 `backend` 及其子目录 `app`, `app/routes`, `app/utils`。
2.  **配置文件创建**：
    *   `requirements.txt`：包含 Flask, SQLAlchemy, PyMySQL, Flask-Migrate 等核心依赖。
    *   `run.py`：应用入口文件。
    *   `app/config.py`：数据库与环境配置。
3.  **核心代码脚手架**：
    *   `app/__init__.py`：初始化 Flask 应用、数据库扩展和注册蓝图。
    *   `app/models.py`：根据报告中的数据库设计，定义 `User`, `Sample`, `Annotation`, `Task`, `Event` 等数据模型。
    *   `app/routes/*.py`：创建 `rumor.py`, `annotate.py`, `user.py` 等空的蓝图文件。

### 第二阶段：前端环境搭建 (Frontend)
1.  **初始化项目**：使用 `pnpm create vite` 创建 Vue3 + TypeScript 项目。
2.  **安装依赖**：安装 `axios`, `vue-router`, `pinia`, `element-plus` (可选，用于快速搭建 UI) 等基础库。
3.  **目录调整**：按照规范创建 `views`, `services` 等目录。
4.  **配置优化**：
    *   配置 `vite.config.ts` 以支持 `@` 路径别名。
    *   封装基础的 Axios 请求拦截器。

### 第三阶段：验证与运行
1.  确保后端可以正常启动并连接（模拟）数据库。
2.  确保前端可以运行并成功构建。
