# 系统功能模块实现流程

系统前端基于 Vue3 + TypeScript 构建单页应用，后端使用 Flask (Python 3.10.6) 提供 RESTful 接口，数据库采用 MySQL。根据开题报告功能划分，系统包含谣言检测与自动粗标注、人机协同标注界面、样本管理平台、事件聚合与相似事件合并、语言标注与导出、多语言支持等模块。以下对每个模块的实现流程进行说明（不涉及具体代码），重点描述前后端交互、模型服务调用和用户操作等：

  • 谣言检测与自动粗标注： 用户在前端提交待检测文本后，Vue3 前端通过 Axios 向后端谣言检测接口（如 /api/rumor_detect）发送请求[1]。Flask 后端接收请求后，将文本传入预训练的谣言检测模型（可部署为独立服务或Python模块），模型输出真实性分类（例如“真实”、“谣言”、“待验证”）。后端将模型预测结果存入数据库的样本表对应记录中，作为“粗标注”标签。此过程可视为系统自动生成初步标签，再由人工复核[2]。检测完成后，后端返回结果给前端，前端页面实时显示判决结果及可信度评分。整个流程示例如图：前端发送文本 → 后端调用模型服务 → 模型返回标签 → 后端保存结果并返回前端。值得注意的是，为实现自动标注与人工审校结合，可先用模型自动标注再交由人工复核[2]，从而形成混合标注流程。
  • 人机协同标注界面： 系统提供交互式的标注UI，前端页面展示自动标注结果供标注员审核和修正[3]。具体流程如下：首先，前端通过接口（如 /api/annotation/tasks）拉取需要标注的样本列表，包括模型预标注的标签和样本内容。页面以表格或卡片形式展示每条样本及其模型预估标签，标注员可在页面上修改标签或补充说明。修改后点击提交时，前端通过 Axios 将标注结果（样本 ID、标注者 ID、最终标签、备注等）POST 到后端接口（如 /api/annotation/submit）。后端接收到标注数据后，在标注记录表中插入新条目，并更新样本表中的最终标签字段；同时可记录人工修正的信息以优化后续模型[3]。整个标注过程支持键盘快捷键和批量操作以提升效率[2]。前后端交互示例：前端加载预标注样本 → 用户修改标签 → 前端提交修改 → 后端保存更新标注。该模块重点在于界面友好、操作流畅，确保标注员快速审核自动标注并纠错[3]。
  • 样本管理平台： 该模块负责对待标注样本的增删改查及状态管理。用户可通过前端界面上传新样本（文本或数据集），或在界面上查看、搜索样本。前端页面向后端发送请求（如 GET /api/samples 列表查询，POST /api/samples 新增样本，PUT/DELETE 对应编辑或删除）[1]。后端对应接口利用 Flask-SQLAlchemy 将样本数据写入 MySQL，并返回操作结果。样本表记录包括文本内容、来源、语言类型、自动标签、所属事件等字段。前端可通过分页和筛选展示样本列表，例如按标签真/假、按时间排序等。新增样本时，也可触发自动检测模块进行初步标注。整个流程示意：前端请求样本列表 → 后端返回数据库查询结果 → 页面显示样本；前端提交新增样本 → 后端存入数据库并返回确认。 样本管理平台为其他模块提供数据基础，确保样本信息可被检索与维护。
  • 事件聚合与相似事件合并： 本模块用于将相关谣言样本归并为事件簇，以便分析。实现流程可包括：后端定时或前端触发式地收集样本文本信息，并调用事件聚类算法（如基于文本相似度或向量聚类的方法）生成事件组。具体可通过 REST 接口（如 /api/events/cluster）将待聚合文本发送给后端，后端使用 NLP 模型计算样本之间相似度，输出事件簇结果。聚类结果在数据库中以“事件表”记录，每个事件包含多个样本（样本表存储一个 event_id 外键指向事件表）。前端在事件聚合界面展示各事件及其样本列表，并允许用户手动合并误分类事件。例如用户可以勾选两个相似事件并点击“合并”，前端向后端发送合并请求，后端更新相应样本的 event_id 或合并事件记录。流程：获取当前事件数据 → 聚合算法分组事件 → 前端展示聚类结果 → 用户选择合并 → 后端更新事件信息。 通过事件聚合，可对谣言进行分组管理，提升数据组织效率。
  • 语言标注与导出： 本模块主要用于对多语言文本进行标注和导出操作。用户可在前端为样本指定语言类型（如中文、英文等），前端提供语言下拉选择或自动检测提示。前端将语言标注通过 API（如 PUT /api/samples/{id}/language）发送给后端，后端更新样本表中的 language 字段。导出方面，前端提供“导出标注”按钮，用户选择需要导出的数据范围后触发导出请求（如 GET /api/export?lang=zh）。后端接到请求后，从数据库查询匹配条件的标注数据，并使用 Python 的 CSV 或 Excel 库生成文件。例如使用 csv.writer 将结果写入内存字符串，再通过 make_response 返回给前端[4]。前端收到响应后弹出文件下载对话框。流程示意：用户在界面标注语言 → 后端更新数据库 → 用户点击导出 → 后端生成CSV/Excel并返回 → 前端下载文件[4]。这种方式实现了标注数据的一键导出，方便后续分析和归档。
  • 多语言支持： 前端界面使用国际化方案，使系统支持中英等多种语言显示。具体可采用 vue-i18n 插件，在前端项目中创建语言文件（如 en.ts、zh.ts），并在 main.ts 中挂载 i18n 实例。页面文本通过 $t('key') 调用对应翻译字符串[5]。用户可以在界面上通过下拉菜单切换语言选项（如示例中包含“中文”、“English”选项[5]），页面内容将动态切换翻译文本。后端方面，确保使用 UTF-8 编码处理多语言文本，无需对接口做特殊语言适配。系统存储多语言样本时会在数据库中记录语言字段，并在检索或导出时根据需求区分。通过前端国际化设计，用户可在同一界面自由切换系统语言，提高可用性[5]。

## 项目推荐目录结构

项目采用前后端分离架构，前端和后端分别作为独立项目管理。推荐目录如下（示例）：

project-root/
├── frontend/ # Vue3 前端项目
│ ├── public/ # 静态资源（index.html 等）
│ ├── src/
│ │ ├── main.ts # 入口文件
│ │ ├── App.vue
│ │ ├── components/ # 通用组件
│ │ ├── views/ # 页面视图组件
│ │ ├── router/ # 路由配置
│ │ ├── store/ # 状态管理（Vuex/Pinia）
│ │ └── services/ # 封装Axios请求的API服务
│ ├── tsconfig.json
│ └── package.json
└── backend/ # Flask 后端项目
├── app/ # 应用包
│ ├── __init__.py # 创建Flask app并注册蓝图
│ ├── models.py # SQLAlchemy模型定义
│ ├── routes/ # 各模块路由（使用Blueprint）
│ │ ├── rumor.py # 谣言检测相关接口
│ │ ├── annotate.py # 标注相关接口
│ │ ├── sample.py # 样本管理接口
│ │ ├── event.py # 事件聚合接口
│ │ ├── export.py # 数据导出接口
│ │ └── user.py # 用户和认证接口
│ ├── utils/ # 工具函数（如模型调用、文本处理）
│ └── config.py # 配置（数据库URI、模型路径等）
├── migrations/ # Flask-Migrate 生成的数据库迁移脚本[6]
├── requirements.txt # Python依赖列表
└── run.py # 应用启动脚本

后端使用 Flask 的 Blueprint 模块化管理各功能路由[7]。app/routes/ 目录下每个文件对应一个功能子模块，例如 rumor.py 负责谣言检测的API。migrations/ 目录用于存放数据库迁移版本文件[8][6]。后端数据库表结构通过 SQLAlchemy 定义在 models.py 中，并通过 Flask-Migrate/Alembic 管理版本迁移[6][8]。前端项目则使用 Vue CLI 标准结构，src/services 中封装Axios请求实例，与后端接口进行通信[1]。
