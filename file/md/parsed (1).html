# 登录与鉴权页面

  • 功能定位：系统所有功能的入口，提供用户登录并获取访问权限（如 JWT 令牌）[1]。
  • 主要内容：用户名/密码输入框、登录按钮、必要的第三方登录选项。
  • 用户操作路径：用户在此输入凭证并提交表单，前端调用后端登录接口进行身份验证，成功后获得 Token 并存储（如 Cookie/LocalStorage）[1]。
  • 前后端交互：前端向后端发送登录请求，后端返回认证信息（Token）。登录成功后，前端使用 Token 访问后续受保护接口。
  • 权限控制：所有用户均可访问该页；登录后根据角色决定后续页面权限。

## 仪表盘（Dashboard）

  • 功能定位：管理员或普通用户登录后的首页，总览系统状态和关键指标[2]。
  • 主要内容：展示统计卡片（如待处理任务数、已标注样本数、事件数量等）、可视化图表（如每日标注量折线图、谣言/非谣言比例图等）[2]。
  • 用户操作路径：用户登录后进入仪表盘，查看概览；可点击卡片或导航跳转到具体功能页面（如点击“待处理任务”卡片进入任务列表）。前端加载时通过接口获取各项统计数据并渲染图表。
  • 前后端交互：前端在加载时调用后端统计接口获取指标数据（如总样本数、分类分布等），并动态生成图表展示[2]。点击卡片时可调用相应路由跳转并获取详细数据。
  • 权限控制：视具体设计而定，一般普通用户可见自己的任务统计；管理员可见全局统计，并可能看到更多运营数据。

## 样本管理页面

  • 功能定位：集中管理系统中的谣言样本（文本、链接等），支持增删改查。
  • 主要内容：样本信息列表（表格），包含样本标题、摘要、来源、上传时间、标注状态等字段；可对样本进行搜索、过滤和分页。工具条提供“新增样本”、“导入数据”等操作按钮。
  • 用户操作路径：用户（通常为管理员或数据管理员角色）进入该页查看样本列表；可以点击“新建/导入”上传样本，或点击某条记录进入样本详情页进行编辑；修改后保存并调用后端接口更新。
  • 前后端交互：页面加载时通过 API 获取样本列表（GET），新增样本时提交表单调用后端创建接口（POST），编辑时调用更新接口（PUT/PATCH），删除时调用删除接口（DELETE）。状态更改后前端刷新列表数据。
  • 权限控制：通常只有管理员或样本管理员可执行新增/删除操作；普通标注员可浏览或查看样本详情，权限可细分为只读或有限编辑。

## 任务分配页面

  • 功能定位：用于将标注任务分配给用户，实现人机协同标注流程的任务调度部分。
  • 主要内容：任务列表（表格），显示任务名称、样本总数、已标注数、分配状态等；支持按项目或事件创建新任务。表单用于配置新任务（选定样本集合、指派标注员/审核员、开启 AI 预标注、多人作业等）[3]。
  • 用户操作路径：管理员或标注管理员进入页面，点击“创建任务”弹出配置表单，填写标注项、分配标注员及数量，选择是否启用 AI 预标注等[4][5]。提交后调用后端接口生成任务，并向标注员推送标注通知。
  • 前后端交互：提交创建任务表单时，前端调用后端接口（POST）创建新任务；页面加载任务列表时调用 GET 接口；对已有任务进行“移交”或“结束”等操作时调用相应接口更新任务状态[6]。
  • 权限控制：仅管理员、标注管理员角色可访问此页面并创建/分配任务[7]。普通标注员和审核员不可创建任务，但可在自己的任务列表中查看受指派的任务。

## 谣言检测与粗标注页面

  • 功能定位：展示系统自动谣言检测和预标注结果，辅助用户初步判别和编辑。
  • 主要内容：输入框或列表形式显示待检测文本及机器判定结果（如“真实/谣言”及置信度）[8]；可选择单条检测或批量导入文本进行自动判别。页面提供模型说明、检测按钮等。
  • 用户操作路径：用户输入或粘贴文本后点击“检测”按钮，前端调用后端 BERT 模型接口，返回“真假”结果及置信度[8]。在列表模式下，用户可批量上传新样本，系统自动标注初步结果。用户可以对自动结果进行确认或修改，然后将样本添加到标注任务中。
  • 前后端交互：前端调用后端文本检测 API（POST 文本内容），后端返回分类结果和置信度[8]。若用户采纳或修改结果，前端发送更新请求保存到样本数据库。生成新样本时还调用导入接口。
  • 权限控制：一般所有登录用户可使用检测功能；但“机器预标注”功能的开启与否可能仅管理员可配置。粗标注结果可供后续标注任务使用。

## 人机协同标注页面

  • 功能定位：标注员执行样本标注的核心界面，结合机器预标注与人工校正。
  • 主要内容：标注任务列表，显示分配给当前用户的所有任务。进入具体任务后，显示单条样本内容（文本或图片）和系统给出的预标注结果（如文本分类标签或实体高亮）。页面提供标注工具（如标签选择、文本高亮工具、复选框等）和提交按钮。界面通常包含“上一个/下一个”导航和标注进度统计。
  • 用户操作路径：标注员在任务列表页选择一个任务后，进入标注作业界面。逐条查看样本内容，参考机器预标注，选择或修改标签等操作。完成一条标注后点击“提交”保存并加载下一条[6]。当所有条目标注完成后，显示任务完成提示，并调用后端更新任务状态。
  • 前后端交互：进入标注界面时，前端通过接口（GET）拉取当前样本数据及预标注结果。每次提交标注结果时，前端调用后端接口（POST/PUT）保存标注数据，并更新该样本/任务的状态[6]。如果启用多人协作，可调用接口移交任务或合并标注。
  • 权限控制：仅分配到该任务的标注员可访问对应标注界面[7]。管理员或标注管理员可查看所有任务标注页面，且可能拥有“审核员”权限。

## 标注审核页面

  • 功能定位：审核员对已标注数据进行复审，确保标注质量。
  • 主要内容：审核任务列表，显示指派给当前审核员的审核作业。进入审核界面后，展示标注员已提交的样本标注结果，并提供审核意见输入框、驳回原因及“通过/不通过”按钮。界面也可统计已审核条目数量和待审核量。
  • 用户操作路径：审核员在列表中选择任务进入审核页面，一条条查看标注结果，点击“通过”或“不通过”，若不通过可填写原因并将样本退回给标注员修改[9][10]。完成所有审核后，提交审核结果更新任务状态。
  • 前后端交互：前端获取审核任务数据（GET），提交审核决定时调用后端接口（POST/PUT）更新样本状态及审核记录[10]。审核完成后，若任务配置为多人审核，前端可调用生成/发布接口生成最终数据集。
  • 权限控制：仅审核员角色可访问该页面[7]。未开启审核功能时，该模块隐藏或不可用。

## 事件聚合页面

  • 功能定位：对标注后的谣言样本进行聚类与合并，将相关样本归为同一“事件”，帮助分析谣言主题[11]。
  • 主要内容：事件列表或可视化聚类图，展示系统自动聚合出的谣言事件（每个事件包含多条相关样本）。可通过列表查看事件名称、样本数量、标签类别等；可通过图表（如力导向图）直观显示事件间关系和内部结构[11]。事件详情页列出该事件包含的所有样本和各自标注，支持人工合并/拆分事件。
  • 用户操作路径：用户打开事件聚合页面查看现有事件，点击某事件进入其详情查看样本。可对事件进行编辑：添加/移除样本、重命名事件、调整事件标签等操作。系统自动聚类结果可作为初始值，用户可人工微调。
  • 前后端交互：前端从后端获取事件聚合数据（GET），包括每个事件的样本列表和标签统计。对事件的任何修改（合并事件、更新标签）由前端调用后端接口（PUT/DELETE/POST）同步更新。可调用后端聚类算法接口重新计算事件聚合。
  • 权限控制：一般所有标注员可查看事件和提交合并建议；仅管理员或审核员可执行事件编辑和最终合并操作。

## 导出与报表页面

  • 功能定位：将标注数据导出为报表或文件，用于离线分析或其他系统使用。
  • 主要内容：导出选项界面，包括数据范围（如所有样本、某任务、某事件）、导出格式（CSV、Excel、JSON等）选择、字段筛选条件、语言等。显示可导出数据量预估，并提供“导出”按钮。可另设制式报表（如按日统计、按标签统计）下载。
  • 用户操作路径：用户在该页设置导出条件（选择时间区间、任务/事件、是否包含机器标签等），点击“导出”后前端向后端请求生成文件或直接获取文件流。导出完成后，前端提示并提供文件下载链接。
  • 前后端交互：前端调用后端导出接口（GET/POST），后端根据条件查询数据并生成文件，返回文件下载地址或直接推送文件流。前端接收后触发下载。
  • 权限控制：通常限管理员或数据分析角色访问；普通标注员可能只能导出其参与标注的数据。

## 多语言支持

  • 功能定位：提供界面文本的多语言切换，满足国际化需求。
  • 主要内容：页面右上角或导航栏中的语言切换器（下拉菜单），列出支持的语言选项（如中文、英文等）。所有页面文本内容通过 Vue-i18n 等国际化方案加载对应语言资源。
  • 用户操作路径：用户在任何页面通过语言切换组件更换界面语言，前端即时更新界面文本显示；并可刷新后仍保持所选语言（存储在用户偏好或 Cookie 中）。
  • 前后端交互：主要在前端实现，无需额外后端接口；语言包在前端项目中维护，使用 vue-i18n 等插件加载资源[12]。必要时，可调用后端获取多语言内容（如标签说明、帮助文档等）。
  • 权限控制：对所有用户开放。

## 系统设置与用户管理

  • 功能定位：管理员配置系统参数、管理用户账号与权限。
  • 主要内容：用户管理子页面：用户列表（表格），包含用户名、角色、状态等，提供新增/编辑/删除用户表单；角色权限设置表单；全局系统配置页（如语言默认值、模型参数）。
  • 用户操作路径：管理员进入用户管理，点击新增按钮填写新用户信息并分配角色，前端调用后端注册接口创建用户。编辑用户时修改其角色和状态。系统设置页可修改系统标题、分页大小等配置，保存时调用接口更新系统配置。
  • 前后端交互：前端通过 API 获取/提交用户及设置数据（GET/POST/PUT/DELETE）。如启用日志审计，前端页面也可请求日志接口。
  • 权限控制：仅超级管理员或系统管理员可访问此模块[7]；普通用户无法进入此页面。
